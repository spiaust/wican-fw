WiCAN PRO v4.50p Full Recovery

Complete ESPTool Web procedure for rebuilding the bootloader, 16 MB partition table, OTA metadata, application, and storage layout.

Important: This is a complete recovery. It erases all existing WiCAN settings and rewrites the full 16 MB flash. Once erasing starts, the WiCAN will not boot normally until programming finishes successfully.

Before starting

  • Use a Windows computer with a current version of Google Chrome or Microsoft Edge. Safari does not support ESPTool Web.
  • Use a known-good USB-C data cable, not a charging-only cable.
  • Keep the vehicle parked throughout the recovery.
  • Make sure the vehicle battery is healthy and the computer will not sleep.
  • Close serial monitors or terminal programs that could be using the WiCAN COM ports.

1. Power the WiCAN PRO

  1. Turn the vehicle completely off.
  2. Plug the WiCAN PRO firmly into the vehicle's OBD-II port.
  3. Confirm that the WiCAN LEDs turn on.
  4. Leave the WiCAN connected to the OBD port throughout the entire recovery.
The USB-C connection alone cannot reliably power the WiCAN PRO. The OBD connection supplies operating power. If the vehicle switches off its OBD port, use accessory mode without starting the engine. Do not proceed with a weak vehicle battery.

2. Connect USB and identify Serial-A

  1. Connect the USB-C data cable to the WiCAN PRO and then to the computer. Avoid a USB hub if possible.
  2. Right-click the Windows Start button and select Device Manager.
  3. Expand Ports (COM & LPT).
  4. Look for the two new CH342/CH343 serial ports created by the WiCAN PRO.
  5. Identify the port labeled Serial-A and note its COM number, such as COM5.

If no new COM ports appear

  • Unplug and reconnect the USB cable.
  • Try another USB data cable or computer USB port.
  • Install the Windows CH342/CH343 driver and reconnect the device.
  • Confirm that no other program has the serial port open.

3. Connect ESPTool Web

  1. In Chrome or Edge, open https://espressif.github.io/esptool-js/.
  2. Set Baudrate to 115200. This is slower but safer for recovery.
  3. Click Connect.
  4. In the browser's serial-port dialog, select the COM port corresponding to Serial-A.
  5. Click Connect in the dialog.
  6. Watch the console and confirm that it identifies an ESP32-S3.
Do not continue if ESPTool reports a different chip. If both COM ports look identical, try one; a failed connection before erasing does not damage the device. Disconnect and try the other port.

4. Erase the damaged flash

  1. Confirm again that ESPTool is connected to an ESP32-S3 through Serial-A.
  2. Click Erase Flash or Erase.
  3. Confirm the operation if prompted.
  4. Do not touch the OBD connector, USB cable, browser, or computer while erasing.
  5. Wait for the console to report that the erase completed successfully.
After the erase succeeds, do not remove power. Complete the programming step before disconnecting the WiCAN.

If erase fails

If you see Failed to connect, Timed out, Serial port disconnected, or Invalid head of packet, reconnect to Serial-A and repeat the erase. Do not continue to programming until erase completes successfully.

5. Add the recovery image

  1. In the Program section, click Choose File or Add File.
  2. Select wican-fw_obd_pro_v450p_full-recovery_16MB.bin.
  3. In the address or offset field next to the file, enter 0x0000.
  4. Use only this one merged recovery file. Do not also add the four individual firmware files.

6. Program the firmware

  1. Recheck the filename and confirm that the address is exactly 0x0000.
  2. Click Program.
  3. Leave the device and computer undisturbed. A full 16 MB write at 115200 baud can take a considerable amount of time.
  4. Do not unplug anything when the progress first reaches 100%.
  5. Wait for a definite completion message such as Finished, Programming complete, Hash verified, or Hard resetting via RTS pin.

7. Disconnect and restart

  1. After programming reports success, click ESPTool's Disconnect button if one is shown.
  2. Wait approximately five seconds.
  3. Unplug the USB-C cable from the WiCAN PRO.
  4. Unplug the WiCAN PRO from the OBD port.
  5. Wait at least 10 seconds.
  6. Reconnect the WiCAN PRO to the OBD port.
  7. Allow 30–60 seconds for the first startup.

8. Confirm the recovery

  1. Look for a Wi-Fi network named approximately WiCAN_xxxxxxxxxx.
  2. Connect using the factory password @meatpi#.
  3. Open http://192.168.0.10/.
  4. Open the WiCAN About tab.
  5. Confirm that the firmware version reports v4.50p.
Because the entire flash was erased, Wi-Fi, vehicle protocol, Home Assistant, webhooks, and power-saving settings must be configured again.

Final checklist

  • WiCAN remained powered through OBD during erase and programming.
  • Serial-A was selected.
  • ESPTool identified an ESP32-S3.
  • Erase completed successfully.
  • The 16 MB merged recovery file was programmed at 0x0000.
  • ESPTool explicitly reported successful completion.
  • USB was disconnected before cycling OBD power.
  • The About page reports v4.50p.

Recovery image layout: bootloader at 0x0000, partition table at 0x8000, OTA metadata at 0xD000, and v4.50p application at 0x10000.
